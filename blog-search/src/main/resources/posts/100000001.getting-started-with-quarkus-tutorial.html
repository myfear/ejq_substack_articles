Quarkus is a modern Java framework designed for cloud-native applications. It offers fast startup times, low memory footprint, and excellent developer experience.

Why Choose Quarkus?

Quarkus brings several advantages to Java development:

  • Fast Startup: Applications start in milliseconds, making them perfect for serverless and containerized environments.

  • Low Memory Usage: Optimized for minimal memory consumption, ideal for microservices.

  • Developer Joy: Hot reload, unified configuration, and excellent tooling make development a pleasure.

Getting Started

To create a new Quarkus project, use the Quarkus CLI:

quarkus create app com.example:my-app \
    --extensions=rest-jackson,hibernate-orm-panache,jdbc-postgresql
cd my-app

This creates a new project with REST API support, Hibernate ORM with Panache, and PostgreSQL database connectivity.

Running the Application

Start the application in development mode:

./mvnw quarkus:dev

The application will be available at http://localhost:8080. Any changes you make to the code will be automatically reloaded thanks to Quarkus's live coding feature.

Next Steps

Now that you have a basic Quarkus application running, you can:

  • Add more extensions for your specific needs

  • Create REST endpoints to expose your API

  • Define entities and repositories using Panache

  • Build and deploy your application

Quarkus makes it easy to build modern, efficient Java applications that are ready for the cloud.
