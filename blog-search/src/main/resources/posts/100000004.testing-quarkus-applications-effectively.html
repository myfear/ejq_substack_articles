Testing is crucial for maintaining code quality and ensuring your application works correctly. Quarkus provides excellent testing support with JUnit 5 and REST Assured.

Unit Testing

Quarkus makes unit testing straightforward. Here's an example of testing a service:

@QuarkusTest
class UserServiceTest {

    @Inject
    UserService userService;

    @Test
    void testCreateUser() {
        User user = new User();
        user.name = "John Doe";
        user.email = "john@example.com";
        
        userService.create(user);
        
        assertNotNull(user.id);
        assertEquals("John Doe", user.name);
    }
}

Integration Testing

For integration tests, use @QuarkusIntegrationTest to test the full application:

@QuarkusIntegrationTest
class UserResourceIT {

    @Test
    void testGetUserEndpoint() {
        given()
          .when().get("/api/users/1")
          .then()
             .statusCode(200)
             .body("name", is("John Doe"));
    }
}

Testing with Testcontainers

Quarkus Dev Services automatically provides test databases, but you can also use Testcontainers for more control:

@QuarkusTest
@Testcontainers
class DatabaseTest {

    @Container
    static PostgreSQLContainer<?> postgres = 
        new PostgreSQLContainer<>("postgres:15")
            .withDatabaseName("testdb");

    @Test
    void testDatabaseConnection() {
        // Your test code here
    }
}

Mocking Dependencies

Use CDI alternatives to mock dependencies in tests:

@QuarkusTest
class ServiceTest {

    @InjectMock
    ExternalService externalService;

    @Test
    void testWithMock() {
        when(externalService.call()).thenReturn("mocked");
        // Your test code
    }
}

Best Practices

Follow these practices for effective testing:

  • Write tests for all business logic

  • Use meaningful test names that describe what is being tested

  • Keep tests independent and isolated

  • Test both success and failure scenarios

  • Use test data builders for complex objects

With Quarkus's testing support, you can write comprehensive tests that give you confidence in your application's correctness.
