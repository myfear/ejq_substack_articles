Hibernate Panache simplifies database persistence in Quarkus applications by providing an active record pattern and repository pattern in one unified API.

What is Panache?

Panache is Quarkus's approach to making Hibernate ORM more productive. It offers two programming models:

  • Active Record Pattern: Entities extend PanacheEntity and include query methods

  • Repository Pattern: Separate repository interfaces extend PanacheRepository

Defining Entities

With Panache, entity definitions are concise and powerful:

@Entity
public class Product extends PanacheEntity {
    
    public String name;
    public BigDecimal price;
    public String category;
    
    public static List<Product> findByCategory(String category) {
        return find("category", category).list();
    }
    
    public static Product findByName(String name) {
        return find("name", name).firstResult();
    }
}

Query Methods

Panache provides several ways to query data:

// Find all
List<Product> all = Product.listAll();

// Find by field
Product product = Product.find("name", "Laptop").firstResult();

// Find with conditions
List<Product> expensive = Product.find("price > ?1", 1000).list();

// Count
long count = Product.count("category", "Electronics");

Transactions

Panache methods are transactional by default. Use @Transactional for custom transaction boundaries:

@Transactional
public void updateProductPrice(Long id, BigDecimal newPrice) {
    Product product = Product.findById(id);
    product.price = newPrice;
    // No need to call persist() - Panache handles it
}

Benefits of Panache

Panache offers several advantages:

  • Less boilerplate code

  • Type-safe queries

  • Automatic transaction management

  • Built-in pagination support

  • Simplified entity definitions

With Panache, you can focus on your business logic rather than writing repetitive persistence code.
